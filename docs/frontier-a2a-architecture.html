② A2A 协议框架

📋 Agent Card（能力发现）

  • 每个 A2A Agent 发布 /.well-known/agent-card.json
  • 描述: name, description, skills[]
  • claw 启动时 GET 拉取一次并缓存
  • 用于生成 function calling 工具定义

📨 Task（任务委派）

  • Client 向 Server POST /a2a/jsonrpc
  • 方法: message/send
  • 参数: 自然语言任务描述
  • 支持 webhook 推送通知

📊 Task Progress（进度追踪）

  • 状态链: submitted → working → completed
  • 流式事件: task / status-update / artifact-update
  • 最终结果通过 artifact 或 status message 返回
  • 格式化为工具输出展示给用户

🛡 零副作用保证

  • 配置缺失 → 跳过，无工具注册
  • agents 为空 → 同上
  • 全部不可达 + failFast=false → 同上
  • MCP / 前端 / agui-server 逻辑完全不变

③ A2A 调用数据流

1
启动加载

claw.exe 读取 .claw/a2a-agents.json，并行拉取所有 Agent Card，缓存到内存。

2
工具注册

为每个可达的 Agent 生成 RuntimeToolDefinition（如 weather_agent），追加到 GlobalToolRegistry。

3
LLM 选择工具

用户提问 → LLM 推理 → 决定调用 weather_agent 工具，参数: {"task": "查询东京天气"}。

4
任务委派

claw 向远程 Agent POST JSON-RPC message/send，携带任务文本。监听流式进度事件。

5
结果返回

格式化为 [Weather Agent] Task abc: submitted → working → completed\nResult: 东京晴天 25°C，经 TCP → SSE 展示给用户。

⑤ 四个集成挂点（main.rs）

挂点位置行为
① 配置加载MCP 工具加载之后读 a2a-agents.json + external-agents.json
② 工具合并GlobalToolRegistry 构建前拉 Agent Card → 生成 handoff tools → 追加到 runtime_tools
③ 执行分发execute_runtime_tool()识别 A2A/外部工具名 → 调 delegate_task / call_external_agent
④ 结果返回tool 返回值格式化进度链 + 结果 → 经 TCP→SSE 展示

⑥ 配置文件

📁 .claw/a2a-agents.json

  • agents[]: id, type:"native", url, enabled
  • webhook: enabled, url, token
  • options: connectTimeoutMs, failFast
  • 缺失时完全跳过（零副作用）

📁 .claw/external-agents.json

  • proxyBaseUrl: 后端代理地址
  • agents[]: id, type:"openai_compatible", name, description
  • 无 api_key（安全隔离）
  • 含 api_key 字段则拒绝加载
