# 🎯 用户操作指南 - 新版 Frontier

## 📦 新版本已完成，包含以下改进

### 改进 1️⃣ 彻底禁用 CronCreate（完全使用 /heartbeat API）
- **问题**: 有时代理使用 CronCreate，有时使用 /heartbeat API，不稳定
- **解决**: 代码层面强制只用 /heartbeat API，禁用 CronCreate
- **效果**: 心跳创建现在 100% 稳定

### 改进 2️⃣ 多个心跳可以并发执行
- **问题**: 第二个心跳显示"已跳过"，无法执行
- **解决**: 修改执行条件，心跳独立运行，不受用户连接影响
- **效果**: 可创建 5 个、10 个心跳同时执行

### 改进 3️⃣ 停止心跳时立即停止报告
- **问题**: 执行"停止所有心跳"后，后台仍在发送报告
- **解决**: 添加停止标志，任务立即停止
- **效果**: 停止时 100% 立即生效，不会有延迟报告

## 🐛 常见问题

### Q: 为什么禁用 CronCreate？
A: CronCreate 是 Windows 计划任务工具，不稳定。/heartbeat 是内置 API，更可靠。

### Q: 旧的 /croncreate 命令还能用吗？
A: 不能。所有旧命令已禁用，只支持 /heartbeat。

### Q: 怎样知道心跳在运行？
A: 在界面或日志中看到:
```
[Heartbeat] Executing task 'check-pc-config': 查看电脑配置
[Heartbeat] Task 'check-pc-config' is stopped, skipping execution
```

### Q: 能同时运行多少个心跳？
A: 理论上无限制，实际受机器性能限制。建议 3-5 个为佳。

### Q: 停止后还有报告说明什么？
A: 表示有任务正在执行中。现在修复了，停止时会立即停止报告。
